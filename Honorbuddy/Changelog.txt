New features:
--------------
  * Core
    - Questbot nolonger automatically mail items after done selling/repairing if the vendor run was initiated through ForceSell/ForceRepair. (HB-1180)

    - Added the following properties to GossipEntry.  	ServerIndex: An index into the total number of possible gossip options, including options not currently displayed and can be used to identify a gossip option in a locale-independent fassion. 	Cost: The cost in copper if gossip option is selected and causes a confirmation dialog to be displayed. 	ConfirmationText: The text shown in confirmation dialog. 	Flags: Gossip option flags. (HB-2278)

  * GarrisonBuddy
    - Added an option to disable opening Salvage items (Bags and Crates) (HB-2230)

    - You may now disable follower missions from the main GSB Config UI. (HB-2237)

    - Added an end of run parking location to the settings, so you can tell the bot where you would like to have your character stop when everything is finished. See the Config UI to set this value. (HB-2239)

    - Added documentation for the mission group API. (HB-2240)

    - Lists in the config UI are now sorted by item name, or comment, as well as disabled state. (HB-2241)

    - Trading post configs are generated for the characters current faction only. This removes a lot of UI bulk. (HB-2256)

    - Will now check if the opening quest for each building has been completed, before attempting work orders. This resolves issues related to not being able to actually place any yet. (HB-2258)

    - Added some extra configuration options for users who have done faction transfers and have the shipment count bug. (HB-2263)

  * Plugin
    - The Buddy Control Panel now works correctly for HonorbuddyDE users. (HB-2244)

    - Buddy Control Panel now provides three exit options: Close Honorbuddy, Logout of Game client, and Exit Game client. (HB-2246)

Api Changes:
--------------
    - Quest profiles now have access to new methods to assist with quest-driven scenarios.  1) IsScenarioCriteriaComplete(int stageIndex, int criteriaIndex); returns true if criteria for given stageIndex is complete.  2) IsScenarioCriteriaComplete(int criteriaIndex); returns true if criteria for current scenario stage is complete.  3) CurrentScenarioStage; returns the current scenario stage number. (HB-1259)

    - Fixed a bug in MerchantFrame that would cause the bot to not purchase items when it is a currency-based purchase, and less than a full stack. (HB-2257)

    - Query.GetNearestMailbox will now only return mailboxes congruent with toon's faction  (HB-1959)
